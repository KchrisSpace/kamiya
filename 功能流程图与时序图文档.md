## 5.1 用户账户模块设计与实现

### 5.1.1 用户注册

#### 流程图

```mermaid
flowchart TD
    A[用户进入注册页面] --> B[填写注册信息<br/>用户名/手机号/密码]
    B --> C{前端验证格式}
    C -->|格式错误| D[显示错误提示]
    D --> B
    C -->|格式正确| E[发送注册请求<br/>POST /users]
    E --> F{检查用户名/手机号<br/>是否已存在}
    F -->|已存在| G[显示"用户名已存在"提示]
    G --> B
    F -->|不存在| H[创建新用户记录]
    H --> I[生成用户ID]
    I --> J[保存用户信息到数据库]
    J --> K[注册成功]
    K --> L[跳转到登录页面]
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant A as API接口
    participant D as 数据库

    U->>F: 填写注册信息
    F->>F: 前端格式验证
    alt 格式错误
        F->>U: 显示错误提示
    else 格式正确
        F->>A: POST /users<br/>{username, phone, password}
        A->>D: 查询用户是否存在
        alt 用户已存在
            D-->>A: 返回已存在
            A-->>F: 返回错误信息
            F->>U: 显示"用户名已存在"
        else 用户不存在
            D-->>A: 返回不存在
            A->>A: 生成用户ID
            A->>D: 插入新用户记录
            D-->>A: 返回创建成功
            A-->>F: 返回用户信息
            F->>U: 显示注册成功
            F->>F: 跳转到登录页面
        end
    end
```

### 5.1.2 用户登录

#### 流程图

```mermaid
flowchart TD
    A[用户进入登录页面] --> B[输入登录标识<br/>ID/手机号/用户名 + 密码]
    B --> C{前端验证格式}
    C -->|格式错误| D[显示错误提示]
    D --> B
    C -->|格式正确| E[判断输入类型<br/>ID/手机号/用户名]
    E --> F[请求获取用户列表<br/>GET /users]
    F --> G[前端匹配用户信息]
    G --> H{用户是否存在}
    H -->|不存在| I[显示"用户不存在"提示]
    I --> B
    H -->|存在| J{密码是否正确}
    J -->|错误| K[显示"密码错误"提示]
    K --> B
    J -->|正确| L{用户角色}
    L -->|管理员| M[存储adminToken<br/>更新Pinia状态]
    L -->|普通用户| N[存储userToken<br/>更新Pinia状态]
    M --> O[跳转到管理后台]
    N --> P[跳转到首页]
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant A as API接口
    participant D as 数据库
    participant S as Pinia Store
    participant L as LocalStorage

    U->>F: 输入登录信息
    F->>F: 前端格式验证
    alt 格式错误
        F->>U: 显示错误提示
    else 格式正确
        F->>F: 判断输入类型<br/>(ID/手机号/用户名)
        F->>A: GET /users
        A->>D: 查询所有用户
        D-->>A: 返回用户列表
        A-->>F: 返回用户列表
        F->>F: 前端匹配用户信息
        alt 用户不存在或密码错误
            F->>U: 显示错误提示
        else 登录成功
            alt 管理员
                F->>L: 存储adminToken
                F->>L: 存储adminUserId
                F->>S: 更新管理员状态
                F->>U: 显示登录成功
                F->>F: 跳转到管理后台
            else 普通用户
                F->>L: 存储userToken
                F->>L: 存储userId
                F->>S: 更新用户状态
                F->>U: 显示登录成功
                F->>F: 跳转到首页
            end
        end
    end
```

## 5.2 甜品展示与查询模块的设计与实现

### 5.2.1 甜品列表的查询

#### 流程图

```mermaid
flowchart TD
    A[用户进入商品列表页面] --> B[前端请求获取商品数据<br/>GET /products_list]
    B --> C[JSON Server返回商品列表]
    C --> D[前端过滤上架商品<br/>status === '1']
    D --> E[显示商品列表]
    E --> F{用户输入搜索关键词<br/>或选择筛选条件}
    F -->|输入搜索关键词| G[防抖处理<br/>300ms延迟]
    G --> H[前端数据过滤<br/>computed属性]
    F -->|选择价格区间| H
    F -->|选择商品类别| H
    H --> I[实时更新商品列表展示]
    I --> F
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant A as API接口
    participant D as 数据库
    participant C as Computed属性

    U->>F: 进入商品列表页面
    F->>A: GET /products_list
    A->>D: 查询所有商品
    D-->>A: 返回商品列表
    A-->>F: 返回商品数据
    F->>F: 过滤上架商品<br/>(status === '1')
    F->>U: 显示商品列表
    
    loop 用户交互
        U->>F: 输入搜索关键词<br/>或选择筛选条件
        F->>F: 防抖处理(300ms)
        F->>C: 触发computed计算
        C->>C: 关键词筛选
        C->>C: 价格区间筛选
        C->>C: 类别筛选
        C-->>F: 返回筛选结果
        F->>U: 实时更新商品列表
    end
```

## 5.3 AI 创意定制模块的设计与实现

### 5.3.1 AI文生图生成

#### 流程图

```mermaid
flowchart TD
    A[用户进入AI定制页面] --> B[选择AI定制模式]
    B --> C[填写需求信息<br/>预算范围/预期时间/备注]
    C --> D[描述梦想中的甜品<br/>输入文本描述]
    D --> E[选择标签<br/>草莓/巧克力/双层等]
    E --> F[点击"生成设计灵感"按钮]
    F --> G{验证输入}
    G -->|描述为空| H[提示输入描述]
    H --> D
    G -->|验证通过| I[调用AI生成接口<br/>模拟生成图片]
    I --> J[生成3张设计方案图片]
    J --> K[显示设计方案预览]
    K --> L{用户操作}
    L -->|不满意| M[点击"再试一次"]
    M --> I
    L -->|满意| N[选择设计方案]
    N --> O[标记选中状态]
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant A as AI接口
    participant S as 状态管理

    U->>F: 进入AI定制页面
    F->>F: 选择AI定制模式
    U->>F: 填写需求信息<br/>(预算/时间/备注)
    U->>F: 输入甜品描述
    U->>F: 选择标签
    U->>F: 点击"生成设计灵感"
    F->>F: 验证输入
    alt 描述为空
        F->>U: 提示输入描述
    else 验证通过
        F->>A: 调用AI生成接口<br/>(模拟)
        A->>A: 生成设计方案
        A-->>F: 返回3张设计图片
        F->>S: 更新designPreviews状态
        F->>U: 显示设计方案预览
        
        alt 用户不满意
            U->>F: 点击"再试一次"
            F->>A: 重新生成
            A-->>F: 返回新设计方案
            F->>U: 更新显示
        else 用户满意
            U->>F: 选择设计方案
            F->>S: 更新selectedDesignIndex
            F->>U: 显示选中状态
        end
    end
```

## 5.4 购物车与交易结算模块的设计与实现

### 5.4.1 购物车管理

#### 流程图

```mermaid
flowchart TD
    A[用户点击"加入购物车"] --> B{检查登录状态}
    B -->|未登录| C[提示登录]
    C --> D[跳转到登录页]
    B -->|已登录| E[获取商品详情<br/>GET /products_list/:id]
    E --> F{检查商品状态}
    F -->|商品不存在| G[显示"商品不存在"提示]
    F -->|商品已下架| H[显示"商品已下架"提示]
    F -->|库存不足| I[显示"库存不足"提示]
    F -->|商品正常| J{检查购物车中<br/>是否已存在该商品}
    J -->|已存在| K[更新商品数量<br/>PUT /cart/:id]
    J -->|不存在| L[添加新商品<br/>POST /cart]
    K --> M[检查库存是否充足]
    L --> M
    M -->|库存不足| N[显示库存不足提示]
    M -->|库存充足| O[更新Pinia Store]
    O --> P[更新localStorage]
    P --> Q[显示"已添加到购物车"提示]
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant S as Pinia Store
    participant A as API接口
    participant D as 数据库
    participant L as LocalStorage

    U->>F: 点击"加入购物车"
    F->>F: 检查登录状态
    alt 未登录
        F->>U: 提示登录
        F->>F: 跳转到登录页
    else 已登录
        F->>A: GET /products_list/:id
        A->>D: 查询商品详情
        D-->>A: 返回商品数据
        A-->>F: 返回商品信息
        F->>F: 检查商品状态
        alt 商品不存在/已下架/库存不足
            F->>U: 显示错误提示
        else 商品正常
            F->>A: GET /cart?user_id=:id
            A->>D: 查询购物车
            D-->>A: 返回购物车列表
            A-->>F: 返回购物车数据
            F->>F: 检查商品是否已存在
            alt 商品已存在
                F->>A: PUT /cart/:id<br/>{quantity: newQuantity}
                A->>D: 更新购物车项
                D-->>A: 返回更新成功
            else 商品不存在
                F->>A: POST /cart<br/>{user_id, product_id, quantity}
                A->>D: 插入购物车项
                D-->>A: 返回创建成功
            end
            A-->>F: 返回操作结果
            F->>S: 更新购物车状态
            F->>L: 更新localStorage
            F->>U: 显示"已添加到购物车"
        end
    end
```

### 5.4.2 订单生成与结算

#### 流程图

```mermaid
flowchart TD
    A[用户点击"去结算"] --> B[获取购物车商品列表]
    B --> C[遍历商品，检查库存<br/>GET /products_list/:id]
    C --> D{所有商品<br/>库存充足?}
    D -->|否| E[显示库存不足提示<br/>提示具体商品]
    E --> A
    D -->|是| F[构建订单数据<br/>包含商品快照]
    F --> G[创建订单<br/>POST /normal_orders]
    G --> H{订单创建成功?}
    H -->|否| I[显示创建失败提示]
    H -->|是| J[遍历订单商品]
    J --> K[扣除库存<br/>PUT /products_list/:id]
    K --> L[清空购物车<br/>DELETE /cart]
    L --> M[更新Pinia Store]
    M --> N[更新localStorage]
    N --> O[显示"订单创建成功"提示]
    O --> P[跳转到订单列表页面]
```

#### 时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端页面
    participant A as API接口
    participant D as 数据库
    participant S as Pinia Store
    participant L as LocalStorage

    U->>F: 点击"去结算"
    F->>S: 获取购物车商品列表
    S-->>F: 返回购物车数据
    
    loop 遍历每个商品
        F->>A: GET /products_list/:id
        A->>D: 查询商品详情
        D-->>A: 返回商品数据
        A-->>F: 返回商品信息
        F->>F: 检查库存
    end
    
    alt 库存不足
        F->>U: 显示库存不足提示
    else 库存充足
        F->>F: 构建订单数据<br/>(包含商品快照)
        F->>A: POST /normal_orders<br/>{订单数据}
        A->>D: 插入订单记录
        D-->>A: 返回订单ID
        A-->>F: 返回订单信息
        
        loop 遍历订单商品
            F->>A: PUT /products_list/:id<br/>{stock: newStock}
            A->>D: 更新商品库存
            D-->>A: 返回更新成功
        end
        
        F->>A: DELETE /cart<br/>(清空购物车)
        A->>D: 删除购物车项
        D-->>A: 返回删除成功
        A-->>F: 返回操作结果
        
        F->>S: 清空购物车状态
        F->>L: 清空localStorage
        F->>U: 显示"订单创建成功"
        F->>F: 跳转到订单列表
    end
```

```mermaid
    F --> H[弹出二次确认框]
    G --> H
    H --> I{用户确认?}
    I -->|取消| J[关闭确认框]
    I -->|确认| K[发送更新请求<br/>PATCH /normal_orders/:id]
    K --> L{操作类型}
    L -->|取消订单| M[更新status='已取消']
    L -->|确认取餐| N[更新status='已完成'<br/>completed_time=当前时间]
    M --> O[更新成功]
    N --> O
    O --> P[刷新订单列表]
    P --> Q[显示操作成功提示]
```
